# 💬 멀티룸 채팅

**RFC 6455 (WebSocket)** 표준을 준수하여 구현한 **실시간 멀티룸 채팅 애플리케이션**입니다.
디스코드(Discord)의 멀티 채널 시스템에서 영감을 받아, 사용자가 여러 채팅방에 동시에 참여하고 탭(Tab)을 통해 자유롭게 대화를 오갈 수 있도록 설계되었습니다.


## ✨ 주요 기능 (Key Features)

### 1. 서버 (Server Side)
* **WebSocket Server:** 'Java-WebSocket' 라이브러리를 확장하여 비동기 통신 서버 구현.
* **동시 접속 (Multi-Presence):** 하나의 클라이언트가 여러 방(`Room`)에 동시에 참여 가능 (`Set<Room>` 구조).
* **객체지향 설계:**
    * `SessionManager`: 클라이언트 연결(Connection) 관리.
    * `RoomManager`: 채팅방 생성, 삭제, 메시지 라우팅 등 비즈니스 로직 전담.
    * `Room`: 개별 채팅방의 상태 및 브로드캐스팅 담당.
* **자동 방 관리:** 인원이 0명이 된 방은 메모리에서 자동으로 삭제하여 리소스 최적화.

### 2. 클라이언트 (Client Side)
* **멀티 탭 UI:** 좌측 사이드바의 탭(Tab)을 클릭하여 참여 중인 방을 자유롭게 전환.
* **방별 메시지 분리:** 수신된 메시지를 방 별로 메모리에 저장하여, 현재 보지 않는 방의 대화가 섞이지 않음.
* **실시간 동기화:** 내가 참여 중인 방에 메시지가 오면 백그라운드에 저장하고, 탭을 클릭했을 때 렌더링.

## 🛠️ 기술 스택 (Tech Stack)

| 구분 | 기술                      | 설명 |
| :--- |:------------------------| :--- |
| **Language** | Java 21                 | 서버 사이드 로직 구현 |
| **Protocol** | WebSocket (RFC 6455)    | 실시간 양방향 통신 |
| **Build Tool** | Gradle     | 의존성 관리 및 빌드 |
| **Library** | Java-WebSocket          | 웹소켓 서버 구현 |
| **Library** | Gson                    | JSON 데이터 파싱 및 직렬화 |
| **Frontend** | HTML5, CSS3, Vanilla JS | 클라이언트 UI 구현 |
